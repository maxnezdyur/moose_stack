# Authoring: Kernels (residual contributions)

Reach for this guide when you need to contribute a term to a PDE residual on a volume, face, node, or point. If the term lives on a Dirichlet/Neumann boundary, see [bc-authoring.md](./bc-authoring.md). If you only need to compute a derived field for output (no residual), see [auxkernel-authoring.md](./auxkernel-authoring.md).

## When to use this (vs alternatives)

Decide on **discretization first**, then **AD vs non-AD**.

1. Standard FE volumetric term (most common case): use **`ADKernel`** (framework/include/kernels/ADKernel.h:24). This is the modern default — write only `computeQpResidual()` returning an `ADReal`; the Jacobian is generated by automatic differentiation. Use a non-AD **`Kernel`** (framework/include/kernels/Kernel.h:15) only if you have a reason (e.g. matching legacy code, performance-tuning a hand-coded Jacobian, or AD propagation is breaking — see pitfalls).
2. FE residual is purely `JxW * value * _test`: derive from **`KernelValue`** (framework/include/kernels/KernelValue.h:21) / **`ADKernelValue`** (framework/include/kernels/ADKernelValue.h:21) and override `precomputeQpValue()`/`precomputeQpResidual()` — saves recomputing the integrand for every test function.
3. FE residual is purely `JxW * vector · _grad_test` (e.g. diffusion): derive from **`KernelGrad`** (framework/include/kernels/KernelGrad.h:21) / **`ADKernelGrad`** (framework/include/kernels/ADKernelGrad.h:21).
4. Time-derivative term in FE: derive from **`TimeKernel`** (framework/include/kernels/TimeKernel.h:18) (non-AD) or **`ADTimeKernel`** (framework/include/kernels/ADTimeKernel.h:33) — gives you `_u_dot` and `_du_dot_du`.
5. Want one source file that compiles into both `Foo` and `ADFoo`: use **`GenericKernel<is_ad>`** (framework/include/kernels/GenericKernel.h:16) with the `usingGenericKernelMembers` macro.
6. Couple a scalar (kappa) variable into an FE residual row, or assemble a scalar-row contribution from element loops: derive from **`KernelScalarBase`** (framework/include/kernels/KernelScalarBase.h:23) (non-AD) or **`ADKernelScalarBase`** (framework/include/kernels/ADKernelScalarBase.h:23).
7. Finite-volume flux (face) integral: **`FVFluxKernel`** (framework/include/fvkernels/FVFluxKernel.h:30). FV-volumetric (cell) integral: **`FVElementalKernel`** (framework/include/fvkernels/FVElementalKernel.h:25). FV time derivative: **`FVTimeKernel`** (framework/include/fvkernels/FVTimeKernel.h:14). FV residuals are *always* AD — there is no non-AD FV kernel base.
8. Linear-system FV (assembled directly into matrix + RHS, no AD): **`LinearFVFluxKernel`** (framework/include/linearfvkernels/LinearFVFluxKernel.h:19) or **`LinearFVElementalKernel`** (framework/include/linearfvkernels/LinearFVElementalKernel.h:19).
9. Discontinuous Galerkin internal-side term: **`DGKernel`** (framework/include/dgkernels/DGKernel.h:18) (non-AD) or **`ADDGKernel`** (framework/include/dgkernels/ADDGKernel.h:14).
10. Hybridized DG: **`HDGKernel`** (framework/include/hdgkernels/HDGKernel.h:17); for IP-HDG schemes derive from **`IPHDGKernel`** (framework/include/hdgkernels/IPHDGKernel.h:19) and provide an `IPHDGAssemblyHelper`.
11. Strongly-imposed point/nodal constraint or reaction (residual added per-node, not integrated): **`NodalKernel`** (framework/include/nodalkernels/NodalKernel.h:18) or **`ADNodalKernel`** (framework/include/nodalkernels/ADNodalKernel.h:19). For Dirichlet on a boundary use a `NodalBC` instead — see [bc-authoring.md](./bc-authoring.md).
12. Distribute a value at a physical point (point source/sink): **`DiracKernel`** (framework/include/dirackernels/DiracKernel.h:32). Override `addPoints()` to declare locations.
13. ODE row driven by a scalar variable (no spatial integration): **`ScalarKernel`** (framework/include/scalarkernels/ScalarKernel.h:14) or **`ADScalarKernel`** (framework/include/scalarkernels/ADScalarKernel.h:17).
14. Internal-interface term coupling two variables across a sideset: **`InterfaceKernel`** (framework/include/interfacekernels/InterfaceKernel.h:34) (with AD variant `ADInterfaceKernel`).

If the residual you want is on an external boundary, you almost certainly want a **`BoundaryCondition`**, not a kernel — see [bc-authoring.md](./bc-authoring.md). If you'd be using a kernel just to set initial values, see [ic-authoring.md](./ic-authoring.md).

## Contract

### **Kernel** (framework/include/kernels/Kernel.h:15)
Purpose: non-AD FE volumetric residual integrated over an element.
- **Required overrides**: `Real computeQpResidual()` — value of the integrand at `_qp` (must include `_test[_i][_qp]` or `_grad_test[_i][_qp]` factor yourself).
- **Optional overrides**: `Real computeQpJacobian()` (diagonal), `Real computeQpOffDiagJacobian(unsigned int jvar)` (couplings to other field vars), `Real computeQpOffDiagJacobianScalar(unsigned int jvar)` (couplings to scalar vars), `RealEigenVector computeQpOffDiagJacobianArray(...)` (array-variable couplings).
- **validParams() additions**: typically `addRequiredCoupledVar`, `addParam<Real>`, `addRequiredParam<MaterialPropertyName>`.

### **ADKernel** (framework/include/kernels/ADKernel.h:24)
Purpose: AD FE volumetric residual — Jacobian generated automatically. The default choice for new FE kernels.
- **Required overrides**: `ADReal computeQpResidual()` — must use `_grad_test`/`_test` and AD-typed couplings to keep AD active.
- **Optional overrides**: rarely needed; you can override `computeQpJacobian()`/`computeQpOffDiagJacobian()` to *replace* AD with hand-coded entries (advanced). Override `dofIndices()` only for unusual mixed-row cases (LM kernels).
- **validParams() additions**: same as `Kernel`.

### **KernelValue** (framework/include/kernels/KernelValue.h:21)
Purpose: residual of the form `JxW * value * _test[_i]`.
- **Required overrides**: `Real precomputeQpResidual()` — return the scalar value (do NOT multiply by `_test`).
- **Optional overrides**: `Real precomputeQpJacobian()` (returns the scalar by which `_phi[_j]` is multiplied; default 0).
- Note: `computeQpResidual` is `final` here — don't override it.

### **KernelGrad** (framework/include/kernels/KernelGrad.h:21)
Purpose: residual of the form `JxW * vector · _grad_test[_i]` (canonical for diffusion-like terms).
- **Required overrides**: `RealGradient precomputeQpResidual()` — return the vector flux (no `_grad_test` factor).
- **Optional overrides**: `RealGradient precomputeQpJacobian()`.

### **TimeKernel** (framework/include/kernels/TimeKernel.h:18)
Purpose: FE residual contributions involving the time derivative.
- **Required overrides**: `Real computeQpResidual()` (inherited from `Kernel`); typically uses `_u_dot[_qp]` and multiplies the Jacobian by `_du_dot_du[_qp]` for the on-diagonal block.
- **Optional overrides**: `computeResidualAdditional()` for advanced post-quadrature contributions.
- Available members: `_u_dot`, `_du_dot_du`.

### **GenericKernel\<is_ad\>** (framework/include/kernels/GenericKernel.h:16)
Purpose: write one templated kernel that compiles to both AD and non-AD via typedefs.
- **Required overrides**: `GenericReal<is_ad> computeQpResidual()` (or whatever the parent demands).
- **Pattern**: declare `template <bool is_ad> class FooTempl : public GenericKernel<is_ad>`, then `typedef FooTempl<false> Foo; typedef FooTempl<true> ADFoo;`. In the .C, `registerMooseObject("App", Foo); registerMooseObject("App", ADFoo);` and instantiate both `template class FooTempl<false>;` and `template class FooTempl<true>;`. Add `usingGenericKernelMembers;` inside the class body so two-phase lookup finds inherited members.
- **validParams() additions**: forward to `GenericKernel<is_ad>::validParams()`.

### **KernelScalarBase** (framework/include/kernels/KernelScalarBase.h:23)
Purpose: FE kernel that *also* contributes to a scalar-variable row (`kappa`) — i.e. assembles cross-terms between a field var and a scalar var while looping over elements.
- **Required overrides**: `Real computeQpResidual()` (field row, from `Kernel`); `Real computeScalarQpResidual()` (scalar row); `Real computeScalarQpJacobian()`, `Real computeQpOffDiagJacobianScalar(jvar)`, `Real computeScalarQpOffDiagJacobian(jvar)`, `Real computeScalarQpOffDiagJacobianScalar(svar)` for full Jacobian.
- **validParams() additions**: typically `addCoupledVar("scalar_variable", ...)`; the parent wires `_kappa_var_ptr`, `_kappa`, `_k_order`.

### **FVKernel** (framework/include/fvkernels/FVKernel.h:32)
Purpose: shared base for all FV kernels. You won't derive from it directly — pick `FVFluxKernel` or `FVElementalKernel`.

### **FVFluxKernel** (framework/include/fvkernels/FVFluxKernel.h:30)
Purpose: face/surface flux contribution in FV (where divergence theorem applied). Always AD.
- **Required overrides**: `ADReal computeQpResidual()` — returns the face flux (sign convention: outward from the elem side; the kernel is called once per face and applied to elem and neighbor with opposite signs by the framework).
- **Optional overrides**: `gradUDotNormal(state, correct_skewness)` for diffusive `∇u·n`; `skipForBoundary(fi)`; `bool avoidBoundary(fi)`; `Moose::FaceArg singleSidedFaceArg(...)` and `elemArg()`/`neighborArg()` to evaluate functors with proper limiters.
- **validParams() additions**: `addRequiredParam<MooseFunctorName>("coeff", ...)` etc.; if you need wider stencils set `params.set<unsigned short>("ghost_layers") = N;` and add a relationship manager (see `FVDiffusion::validParams`).

### **FVElementalKernel** (framework/include/fvkernels/FVElementalKernel.h:25)
Purpose: cell-volume integral terms in FV (sources, reactions, time derivatives). Always AD.
- **Required overrides**: `ADReal computeQpResidual()`. There's effectively one quadrature point at the centroid; `_qp = 0`.
- **Optional overrides**: rarely; do NOT override `computeResidual`/`computeJacobian` — they have framework-internal logic.
- Available members: `_u` (`ADVariableValue`), `_u_functor`, `_current_elem`, `_q_point`.

### **FVTimeKernel** (framework/include/fvkernels/FVTimeKernel.h:14)
Purpose: FV time-derivative term. Provides `_u_dot` (AD).
- **Required overrides**: `ADReal computeQpResidual()` — by default returns `_u_dot[_qp]`; override only for non-trivial mass terms (e.g. density-weighted).

### **LinearFVKernel** (framework/include/linearfvkernels/LinearFVKernel.h:24)
Purpose: shared base for FV kernels that contribute directly to a linear system's matrix and RHS (no AD). Don't derive directly.

### **LinearFVFluxKernel** (framework/include/linearfvkernels/LinearFVFluxKernel.h:19)
Purpose: face-flux contribution to a pre-assembled linear system.
- **Required overrides**: `Real computeElemMatrixContribution()`, `Real computeNeighborMatrixContribution()`, `Real computeElemRightHandSideContribution()`, `Real computeNeighborRightHandSideContribution()`, `Real computeBoundaryMatrixContribution(const LinearFVBoundaryCondition &)`, `Real computeBoundaryRHSContribution(const LinearFVBoundaryCondition &)`.
- **Optional overrides**: `Real computeBoundaryFlux(...)`.

### **LinearFVElementalKernel** (framework/include/linearfvkernels/LinearFVElementalKernel.h:19)
Purpose: cell-volume contribution to a pre-assembled linear system.
- **Required overrides**: `Real computeMatrixContribution()`, `Real computeRightHandSideContribution()`.

### **DGKernelBase** (framework/include/dgkernels/DGKernelBase.h:32)
Purpose: shared base for DG and ADDG; don't derive directly.

### **DGKernel** (framework/include/dgkernels/DGKernel.h:18)
Purpose: non-AD residual on internal element sides for DG discretizations.
- **Required overrides**: `Real computeQpResidual(Moose::DGResidualType type)`, `Real computeQpJacobian(Moose::DGJacobianType type)`. `type` selects `Element` vs `Neighbor` for residual; element-element / element-neighbor / neighbor-element / neighbor-neighbor for Jacobian.
- **Optional overrides**: `Real computeQpOffDiagJacobian(...)`, `precalculateQpResidual/Jacobian/OffDiagJacobian` for QP-precomputation hooks.
- For AD: derive from **`ADDGKernel`** (framework/include/dgkernels/ADDGKernel.h:14) and override only `ADReal computeQpResidual(Moose::DGResidualType type)`.

### **HDGKernel** (framework/include/hdgkernels/HDGKernel.h:17)
Purpose: hybridized DG kernel base. Inherits from `Kernel` so it sits in the standard kernel warehouse but adds side-loop hooks.
- **Required overrides**: `void computeResidualOnSide()`, `void computeJacobianOnSide()`. `computeQpResidual()` is intentionally a fatal error — don't.
- **Optional overrides**: `computeResidualAndJacobianOnSide()`.

### **IPHDGKernel** (framework/include/hdgkernels/IPHDGKernel.h:19)
Purpose: interior-penalty hybridized scheme.
- **Required overrides**: `IPHDGAssemblyHelper & iphdgHelper()` returning the helper that holds the actual quadrature math. Most physics lives in the helper, not the kernel.

### **NodalKernelBase** (framework/include/nodalkernels/NodalKernelBase.h:24)
Purpose: shared base; don't derive directly.

### **NodalKernel** (framework/include/nodalkernels/NodalKernel.h:18)
Purpose: non-AD residual added per-node (no integration). Block- and boundary-restrictable.
- **Required overrides**: `Real computeQpResidual()`.
- **Optional overrides**: `Real computeQpJacobian()` (defaults to 1), `Real computeQpOffDiagJacobian(unsigned int jvar)`.
- For AD: **`ADNodalKernel`** (framework/include/nodalkernels/ADNodalKernel.h:19) — override only `ADReal computeQpResidual()`.

### **DiracKernelBase** (framework/include/dirackernels/DiracKernelBase.h:26)
Purpose: shared base for Dirac kernels.
- **Required overrides**: `void addPoints()` — call `addPoint(elem, p)` or `addPoint(p)` for each source location. Implement once per object.

### **DiracKernel** (framework/include/dirackernels/DiracKernel.h:32) (templated as `DiracKernelTempl<Real>` / `<RealVectorValue>`)
Purpose: distribute a value at physical points by multiplying by shape functions.
- **Required overrides**: `Real computeQpResidual()`, `void addPoints() override`.
- **Optional overrides**: `Real computeQpJacobian()`, `Real computeQpOffDiagJacobian(unsigned int jvar)`.

### **ScalarKernelBase** (framework/include/scalarkernels/ScalarKernelBase.h:19)
Purpose: shared base for ODE/scalar kernels.
- **Required overrides**: `void reinit()`.

### **ScalarKernel** (framework/include/scalarkernels/ScalarKernel.h:14)
Purpose: non-AD ODE row.
- **Required overrides**: `Real computeQpResidual()`, `Real computeQpJacobian()` (both `mooseError` if not overridden).

### **ADScalarKernel** (framework/include/scalarkernels/ADScalarKernel.h:17)
Purpose: AD ODE row.
- **Required overrides**: `ADReal computeQpResidual()`. Jacobian generated by AD.

### **InterfaceKernelBase** (framework/include/interfacekernels/InterfaceKernelBase.h:24)
Purpose: shared base; don't derive directly.

### **InterfaceKernel** (framework/include/interfacekernels/InterfaceKernel.h:34) (`InterfaceKernelTempl<Real>` / `<RealVectorValue>`)
Purpose: residual on a sideset that couples a primary variable on the elem side with a `neighbor_var` on the neighbor side.
- **Required overrides**: `Real computeQpResidual(Moose::DGResidualType type)`. `type` is `Element` or `Neighbor` — return the contribution to the corresponding test space.
- **Optional overrides**: `Real computeQpJacobian(Moose::DGJacobianType)`, `Real computeQpOffDiagJacobian(...)`, `initQpResidual/Jacobian/OffDiagJacobian` hooks.
- **validParams() additions**: typically `addRequiredCoupledVar("neighbor_var", ...)`. For AD use `ADInterfaceKernel` (framework/include/interfacekernels/ADInterfaceKernel.h).

## Coupling & material properties

- **Coupled values**: non-AD path uses `coupledValue("v")` returning `const VariableValue &`, `coupledGradient("v")` returning `const VariableGradient &`, `coupledDot("v")` returning `const VariableValue &`. AD path uses `adCoupledValue("v")`, `adCoupledGradient("v")`, `adCoupledDot("v")` returning `ADVariableValue` / `ADVariableGradient`. In an AD kernel, *always* prefer the `adCoupled...` versions — using non-AD `coupledValue` in an AD residual silently strips derivatives w.r.t. that variable.
- **Material properties**: `getMaterialProperty<T>("name")` for non-AD (`const MaterialProperty<T> &`). `getADMaterialProperty<T>("name")` for AD (`const ADMaterialProperty<T> &`). Generic templated form: `getGenericMaterialProperty<T, is_ad>("name")` — used inside `GenericKernel<is_ad>` templates.
- **Stateful**: `getMaterialPropertyOld<T>("name")` and `getMaterialPropertyOlder<T>("name")` for previous-step values; mark them stateful by declaring them on the producing `Material` side.
- **Functors**: in FV (and increasingly elsewhere) prefer `getFunctor<ADReal>("name")` returning a `Moose::Functor<ADReal> &`; evaluate with `_coeff(elemArg(), state)` / `_coeff(face_arg, state)` (see `FVDiffusion`). Functors gracefully handle face/elem/qp evaluation with limiters and skew correction.
- **Off-diagonal Jacobian (non-AD only)**: if your residual depends on coupled variable `v`, you must override `computeQpOffDiagJacobian(unsigned int jvar)` and check `if (jvar == _v_var)` to add `∂R/∂v`. Forgetting this gives an incorrect Newton step (often manifests as poor convergence or wrong sensitivities). With AD, off-diag is automatic — that's the headline reason ADKernel is the default.
- **Off-diagonal scalar coupling**: override `computeQpOffDiagJacobianScalar(jvar)` if the residual depends on a scalar variable (non-AD).
- **AD pitfalls specific to kernels**:
  - Mixing in any non-AD coupled value, non-AD material property, or `_var.value()` cuts the AD chain — that variable's derivatives go to zero in the Jacobian.
  - Inside a `GenericKernel<is_ad>` template, use `this->template getGenericMaterialProperty<T, is_ad>(...)` and `this->template coupledGenericValue<is_ad>(...)`. Dropping the `template` keyword fails to compile under GCC.
  - In an AD residual, multiplying by `_JxW[_qp]` is *not* needed (the framework does it); but in `KernelGrad`/`KernelValue` precompute methods, the framework also handles `JxW` and `_test` — return only the kernel-specific value/vector.

## Registration & build

- In the `.C` file, after `#include`, call `registerMooseObject("YourAppName", ClassName);` exactly once per concrete class. For `App`-bundled physics modules use the module's app name (e.g. `"HeatTransferApp"`). The framework's own kernels register with `"MooseApp"`.
- AD/non-AD twin pattern: in the header, write `template <bool is_ad> class FooTempl : public GenericKernel<is_ad>` (or directly `Kernel`/`ADKernel` parents picked by `std::conditional`); add `typedef FooTempl<false> Foo;` and `typedef FooTempl<true> ADFoo;`. In the `.C`, `registerMooseObject("...", Foo); registerMooseObject("...", ADFoo);` and emit explicit instantiations: `template class FooTempl<false>; template class FooTempl<true>;`. See `framework/src/kernels/MatReaction.C` for a complete example.
- File locations: framework kernels live in `framework/include/kernels/*.h` + `framework/src/kernels/*.C` with matching basename. Module kernels mirror this under `modules/<module>/include/kernels/` + `modules/<module>/src/kernels/`. FV kernels live under `fvkernels/`, DG under `dgkernels/`, etc. Subdirectories under `include/kernels/` are allowed; the build system globs recursively.
- `.mk` wiring: the default `Makefile` globs `src/**/*.C` recursively, so adding a new `.C` to an existing kernel directory is picked up with no `.mk` edit. Only if you place files in a non-standard location do you need to touch the app `.mk`.

## Minimal scaffold

Compilable `ADKernel` example: a parameterized reaction term `−L·u` reading an AD material property.

```cpp
// modules/myapp/include/kernels/ADMyReaction.h
#pragma once

#include "ADKernel.h"

class ADMyReaction : public ADKernel
{
public:
  static InputParameters validParams();

  ADMyReaction(const InputParameters & parameters);

protected:
  virtual ADReal computeQpResidual() override;

  /// Reaction rate L, read from a Material
  const ADMaterialProperty<Real> & _rate;
};
```

```cpp
// modules/myapp/src/kernels/ADMyReaction.C
#include "ADMyReaction.h"

registerMooseObject("MyApp", ADMyReaction);

InputParameters
ADMyReaction::validParams()
{
  InputParameters params = ADKernel::validParams();
  params.addClassDescription("Reaction kernel: residual = -L * u * test, with L from a material property.");
  params.addRequiredParam<MaterialPropertyName>("rate", "Reaction rate material property L");
  return params;
}

ADMyReaction::ADMyReaction(const InputParameters & parameters)
  : ADKernel(parameters),
    _rate(getADMaterialProperty<Real>("rate"))
{
}

ADReal
ADMyReaction::computeQpResidual()
{
  return -_rate[_qp] * _u[_qp] * _test[_i][_qp];
}
```

## Common pitfalls

- **Wrong test-function factor**. `Kernel`/`ADKernel` integrands must be multiplied by `_test[_i][_qp]` (or `_grad_test[_i][_qp]` after integration by parts) — *you* write the factor. Forgetting it produces an integral of just the strong-form quantity, not a residual. Use `KernelValue`/`KernelGrad` to have the framework apply the factor for you.
- **Non-AD couplings in an AD kernel silently kill derivatives**. Inside `ADKernel::computeQpResidual`, every dependency must come from `adCoupledValue`/`adCoupledGradient`/`getADMaterialProperty`. If you read `coupledValue("v")` (returns plain `Real`), the Jacobian column for `v` will be wrong and Newton will stall — often invisibly because the residual still looks plausible.
- **Off-diagonal Jacobian missing for non-AD couplings**. A non-AD `Kernel` whose residual depends on a coupled variable `v` *must* override `computeQpOffDiagJacobian(jvar)` and return `∂R/∂v` when `jvar == _v_var`. Symptom: the Newton solver's quadratic convergence degrades to linear, or solves diverge under tight tolerances. Use `-snes_check_jacobian` to catch this.
- **Time-derivative kernels need `TimeKernel`/`ADTimeKernel`**. Putting `_u - _u_old` math in a plain `Kernel` works mechanically but bypasses the time-integrator wiring — the term won't be re-evaluated correctly under non-trivial integrators (BDF2, Crank-Nicolson, etc.). Derive from `TimeKernel`/`ADTimeKernel` and use `_u_dot[_qp]` instead.
- **FV is always AD; FE is opt-in**. `FVFluxKernel`/`FVElementalKernel` `computeQpResidual` returns `ADReal` and there's no non-AD variant. Don't try to mirror an `FVKernel`/`FVADKernel` split — it doesn't exist. Do the opposite: when porting an FV residual into FE, switch to `Real` and decide whether to use `ADKernel` or `Kernel`.
- **`GenericKernel<is_ad>` template lookup gotchas**. Inside the templated class body, two-phase name lookup hides inherited members (`_qp`, `_u`, `_test`, `coupled`, `getGenericMaterialProperty`). Add `usingGenericKernelMembers;` and prefix dependent calls with `this->template`. Forgetting either causes "not declared in this scope" or "expected primary-expression before '>'" errors that look unrelated.
- **DG/Interface kernels: residual sign and `type` parameter**. `computeQpResidual(Moose::DGResidualType type)` is called *twice* per face, once with `Element` and once with `Neighbor`. You must return the contribution to the *appropriate* test space — re-using the same expression for both halves typically loses a sign and breaks symmetry of the bilinear form. Read an existing example (e.g. `DGDiffusion`) before writing one from scratch.
- **DiracKernel `addPoints()` must be deterministic and idempotent per timestep**. The framework caches point-to-element mappings; if `addPoints()` returns different points across calls within a step you'll see lost contributions or asserts. For moving sources, set `allow_moving_sources = true` in `validParams`.
